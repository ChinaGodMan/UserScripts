# Github Email Finder (Alpha)

該腳本用於在 GitHub 用戶主頁自動提取用戶公開的提交（PushEvent）中的郵箱地址(如果沒有公開的郵箱地址,)，並將郵箱信息插入到頁面中，同時提供一鍵複製郵箱到剪貼板的功能。

## 功能概述

-   檢查當前頁面是否已有郵箱信息展示。
-   如果沒有，從 GitHub API 獲取用戶公開事件數據，提取最近的 PushEvent 中的提交郵箱。
-   將提取到的郵箱插入到用戶資料頁的聯繫信息區域。
-   在郵箱旁邊插入一個剪貼板圖標，點擊即可將郵箱複製到剪貼板。
-   複製成功後，剪貼板圖標會替換為打鉤圖標，提示複製成功。

## 使用說明

1. 確保配置了有效的 GitHub 個人訪問令牌（`GITHUB_TOKEN`）[此處](https://github.com/settings/tokens/new?description=Github%20Email%20Finder&scopes=repo)生成新令牌，以避免 API 請求限制。

## 注意

-   此腳本獲取的並不是準確的郵箱地址，而是提交者郵箱地址，可能不是用戶註冊郵箱。(比如合併請求中的郵箱地址,那麼顯示的郵箱地址就是合併請求發起者的郵箱地址)
-   此腳本目前處於測試階段，存在一些問題。
